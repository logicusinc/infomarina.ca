Ajouts et améliorations

  • Le logiciel InfoMarina est maintenant en 64 bits.
  • Si l'option de Gérer le signalement des clients qui ont quittés est activée dans les configurations, vous constaterez sous l'onglet Plan, du menu principal, liste des emplacements Non libérés:
    • le bouton Libérer a été remplacé par le bouton Libérer un. Le bouton Libérer un permet de libérer le seul élément sélectionné de la liste;
    • le bouton Libérer tous a été ajouté. Il permet de libérer tous les éléments de la liste;
    • le bouton Sélectionner a été remplacé par le bouton Imprimer la liste. Le bouton Imprimer la liste permet d'imprimer la liste des occupants qui devraient avoir quittés;
    • le bouton Sélectionner n'est plus utile, car c'est la sélection par le déplacement de la souris sur les emplacements de la liste qui permet d'afficher l'emplacement sur le plan.
  • Il est maintenant possible d'ajouter une ligne de textes dans une facture. Pour utiliser cette fonctionnalité, il faut créer un produit avec le code TEXTE. Ensuite lors de la création de la facture il suffit d'ajouter une ligne avec le produit utilisant le code TEXTE pour pouvoir ajouter du texte à votre facture. Vous pouvez ajouter autant de ligne de texte que vous le désirez.
  • Il est maintenant possible, lors de départ temporaire, d'utiliser un nouveu bouton Date de fin qui va ajouter la date de terminaison de contrat comme date de fin du départ temporaire.
  • Il est maintenant possible, d'utiliser le clic gauche de la souris pour sélectionner un produit lors de la création d'une facture.
  • Il est maintenant possible, de créer un client comptoir par défaut afin d'augmenter la rapidité lors de la facturation au comptoir. Pour ajouter un client par défaut il faut aller dans le menu Outils, Paramètres, Configuration générale, Comptabilité, Onglet Facturation sélectionné dans la liste le compte client à utiliser pour les factures comptoir. Un bouton Facture comptoir a été ajouté dans la fenêtre de facturation
  • Ajout de la langue English (US) afin de permettre la gestion des adresses utilisant le Zip code comme les États-Unis au lieu de l'anglais du canada pour les codes postaux canadiens.
  • En langue anglaise, correction du mot Gazoline avec un z par Gasoline avec un s dans la définition d'un produit.
  • En langue anglaise, correction du mot Check par le mot Cheque dans les types de paiements.
  • En langue anglaise, correction du mot Contrat par le mot Contract sur le carton de visite.
  • En langue anglaise, changer du mot Arrival par le mot Motor sur le carton de visite.
  • En langue anglaise, changer du mot CONTRAcT par le mot CONTRACT sur le résumé du contrat.
  • Diverses petites corrections et modifications pour améliorer le logiciel InfoMarina.

Ajout - Départ temporaire (coupon de tirage).

  • Il est maintenant possible de créer et d'imprimer des coupons pour chaque tranche de 24 heures d'un départ temporaire relié à un contrat. Les coupons permettent par la suite d'effectuer un ou des tirage(s) afin de remercier la clientèle qui avise la marina de leur départ. Un tirage est identifié par son numéro de tirage. Il est possible de terminer un tirage pour supprimer les participations. Un nouveau tirage est automatiquement créé avec un nouveau numéro. Le numéro du tirage est compris dans l'idenfifant du numéro de coupon. Un coupon comprend le numéro de coupon, le nom du client, le nom du bateau, la date de départ temporaire et la date d'arrivée.

    Si un départ temporaire est modifié (abrégé en nombre de jours) le nombre de participations (coupon) sera automatiquement diminué si la liste des coupons n'a pas été imprimée. Si la liste a déjà été imprimée, rien n'est effectué. Le nombre de coupons est alors plus grand que le nombre de coupons prévus.

    Si le départ temporaire est modifié (augmenté en nombre de jours) le nombre de participations (coupon) sera automatiquement augmenté si la liste des coupons n'a pas été imprimée.Si la liste a déjà été imprimée, le nombre de participations (coupons) est augmenté et un message est affiché afin d'aviser qu'un nombre de coupons additionnels n'a pas été imprimé.

    La gestion des coupons et des tirages est disponible sous le menu «Outils», «Tirage».

    Les tirages peuvent être effectués manuellement à partir des coupons imprimés ou à partir d'un bouton «Effectuer un tirage» de la fenêtre. Le gagnant est sélectionné aléatoirement à partir des participations (coupons) de la liste.

Ajout - «Gestion des espaces minimaux entre deux bateaux».

  • Sous le menu «Outils», «Paramètres», «Configuration générale», onglet « Général », «Bateaux» un nouveau champ « Espace minimale entre les bateaux » afin de permettre la gestion d'espace minimale à respecter. De plus, pour permettre une gestion efficace il faut ajouter les quais adjacents, sous le menu «Gestion, Emplacements » nouvelles colonnes «Quai adjacent». Suite à la sélection d'un quai adjacent, la largeur du quai dans la colonne «Quai adjacent largeur» est automatiquement affichée dans la colonne. Lors de l'ajout d'un bateau à un quai, si espace minimale entre les bateaux n'est pas respecté, un message sera affiché.

Ajout - Onglet « Visiteurs ».

  • Une nouvelle colonne «Type» avec les codes V, C, E. Les codes sont V pour Visiteur, C pour Contrat et E pour Entreposage. Un client peut avoir un type unique ou composé. Exemple: Un client qui est ou a été sous contrat pour la location d'un quai aura le type C et s'il a aussi été sous contrat pour l'entreposage il aura le type CE. Alors s'il a déjà été visiteur il aura le type VCE. Ainsi il sera plus facile d'identifier le type de client de la liste.

Ajout - Onglet «Facturation».

  • Sous le bouton «Sélectionner» une nouvelle colonne «Cellulaire» a été ajoutée afin de permettre la recherche par numéro de cellulaire.

Ajout – Z de caisse.

  • Nous avons ajouté les numéros de compte grand livre aux codes de produits et/ou de services dans le rapport Z de caisse. De plus, à la fin du Z de caisse, un sommaire de comptes a été ajouté. Le sommaire affiche les totaux par compte. La description des comptes doit être ajoutée manuellement à partir du menu «Outils», «Paramètres», «Configuration générale», «Comptabilité», «Description no. G/L» si le module «Communication ACOMBA» n'est pas activé. Sinon l'utilisation du nouveau bouton «Importer les descriptions d'ACOMBA» permet l'importation de toutes les descriptions de comptes inclus dans la liste à partir des descriptions utilisées dans ACOMBA. Les descriptions de comptes utilisées dans le Z de caisse seront donc les mêmes.

Ajout - Impression des clients qui utilisent les quais des départs temporaires (espaces libres).

Deux nouvelles colonnes nommées «Visiteur de remplacement», «Nombre de jours de remplacement» ont été ajoutée au rapport.

Ajout - Impression «Liste des emplacements / occupants».

  • Sous le menu «Consultation», «Emplacements», «Liste des emplacements / occupants» il est possible d'imprimer la liste des emplacements avec les occupants pour une période donnée. Pour afficher des éléments dans la liste, il fallait que la date de début et la date de fin de la réservation et/ou du contrat entrent dans la période de la date de début et de la date de fin du filtre. Une nouvelle case à cocher a été ajoutées afin de permettre d'afficher dans la liste toutes les réservations et les contrats qui chevauche aussi la période du filtre. Exemple : sans la case cochée date de début du 1 juillet au 31 juillet 2014. Seront affichés dans la liste tous les réservations et/ou contrats qui ont une date de début et une date de fin à l'intérieure de la période du 1 au 31 juillet 2014. Exemple : avec la case cochée date de début du 1 juillet au 31 juillet 2014. Seront affichés dans la liste tous les réservations et/ou contrat qui ont une date de début et une date de fin à l'intérieure de la période du 1 au 31 juillet 2014 et aussi toutes les réservations et/ou contrats qui (chevauche) commence dans de la période du 1 au 31 juillet 2014 et aussi toutes les réservations et/ou contrats qui (chevauche) termine dans l'intérieure de la période du 1 au 31 juillet 2014.

Ajout - Aide à la gestion des villes (Inclus les villes pour les provinces: Québec et Ontario)

  • Une liste de villes du Québec et de l'Ontario a été ajoutée afin de faciliter la saisie d'un nom de ville pour un client. En sélectionnant une ville dans la liste, lors de la modification et/ou de l'ajout d'un nouveau client vous uniformiserez le nom de la ville afin d'éviter d'avoir dans vos données une ville écrite de multiples façons. Cela facilite la gestion, diminue les erreurs et augmente l'efficacité des rapports statistiques.

  • Pour activer cette gestion il faut cocher l'option "Gestion automatisée des noms de villes, provinces, pays et codes postaux" sous le menu "Outils", "Paramètres", "Configuration générale" onglet "Général". De plus, il est possible lorsque cette option est activée de choisir si la sauvegarde des données (villes, provinces, pays) doit être TOUT EN MAJUSCULES ou Majuscules et minuscules.

  • La gestion individuelle des (villes, provinces, pays et codes postaux) est disponible sous le menu "Client", "Automatisation des adresses", option "Villes, provinces et pays" ou option "Codes postaux". Il est possible d'ajouter , modifier, supprimer les villes, provinces, pays et codes postaux à partir de cette gestion. Prendre note que si vous supprimez un pays toutes les provinces, villes et codes postaux de ce pays seront aussi supprimés. Si vous supprimez une province toutes, les villes et codes postaux de cette province seront aussi supprimés. Si vous supprimez une ville, tous les codes postaux de cette ville seront aussi supprimés.

  • Pour gagner en efficacité et en rapidité, lors de la saisie des informations sur un client, si vous entrez le code postal, les informations sur la ville, province et pays sont automatiquement ajoutées. Alors, demander toujours le code postal du client avant de lui demander sa ville. Ainsi lorsque la ville sera automatiquement affichée vous pourrez toujours confirmer si la ville incluse dans INFOMARINA correspond bien à sa ville réelle.
  • Pour les utilisateurs du logiciel ACOMBA avec INFOMARINA il est important de faire la mise à jour des clients ACOMBA suite aux changements ou corrections des villes afin que les deux logiciels possèdent les mêmes informations. Pour mettre à jour les clients ACOMBA via INFOMARINA il faut utiliser le menu dans INFOMARINA "Facturation", "Acomba", "Mise à jour des clients"
  • Villes non répertoriées - Il est possible de valider les villes non répertoriées de votre base de données clients en utilisant le bouton "Villes non répertoriées" inclus dans la fenêtre "Gestion des villes, provinces et pays" du menu "Client", "Automatisation des adresses", option "Villes, provinces et pays". Ce bouton vous permet d'uniformiser l'ensemble des villes de votre base de clients. Les clients avec des villes non conformes ( nom différent que le nom inclus dans la liste des villes) seront automatiquement affichés dans la liste. Ensuite en cliquant sur le bouton (OEIL) "Visualiser/Modifier la fiche client" ou en double cliquant sur un client de la liste, la fenêtre de la fiche client sera ouverte. Pour mettre, individuellement, ajour la ville selon le code postal de la fiche client il suffit de cliquer sur le bouton à droite du code postal (2 FLÈCHES) "Assigner la l'adresse suggérée (VILLES, PROVINCES,PAYS)". Par la suite en cliquant sur le bouton "OK" la fenêtre sera fermée et le client va disparaitre de la liste des villes non répertoriées. Pour mettre, en lot, ajour les villes selon le code postal des fiches clients il suffit de sélectionner avec le bouton "CRT" ou "SHIFT" l'ensemble des clients de la liste pour lesquels vous voulez mettre à jour leur ville. Ensuite sur le bouton de (2 FLÈCHES) "Assigner la l'adresse suggérée (VILLES, PROVINCES,PAYS)". Par la suite tous les clients vont disparaitre de la liste des villes non répertoriées.

Ajout - Arrondissement des paiements comptants (Retrait de la pièce de 1 cent, Canada seulement)

À venir lorsque disponible via ACOMBA, cette gestion permettra le transfert des montants d'arrondissements dans le compte spécifique à l’arrondissement (cents) pour le compte recevable ACOMBA.

Ensuite si vous cliquez sur le bouton "OK" des validations seront effectuées afin de valider si la permutation peut avoir lieu. Si ce n'est pas possible, un message sera affiché. Sinon le message "Opération réussie !" est affiché. Si une réservation a été effectuée dans la période d'un départ temporaire d'un des deux contrats des emplacements sélectionnés, la ligne affichant l'information de la réservation est automatiquement affichée en rouge. Il est alors impossible de faire cette permutation.
